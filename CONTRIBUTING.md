# Contribuindo para Tech 4 Dev

Primeiramente, agradecemos por considerar contribuir para o Tech 4 Dev! É pessoas como você que fazem a comunidade de código aberto um lugar tão fantástico para aprender, inspirar e criar. Todos os tipos de contribuições são incentivadas e valorizadas.

Aqui estão algumas diretrizes que pedimos que você siga para que possamos ter um processo de contribuição consistente e eficiente.

## Como Contribuir

1. **Faça um fork do repositório**
   Comece fazendo um fork do projeto para a sua conta no GitHub. Isso lhe dará uma cópia própria que você pode modificar à vontade.

2. **Clone o repositório**
   Clone o projeto para o seu computador local para começar a trabalhar nele.

   ```sh
   git clone https://github.com/LKSFerreira/tech4dev.git
   cd tech4dev
   ```

3. **Crie uma Branch**
   Sempre crie uma branch para cada nova funcionalidade ou correção.

   ```sh
   git checkout -b feature/NomeDaSuaFeature
   ```

4. **Faça suas alterações**
   Implemente suas alterações e faça commits com mensagens claras e informativas.

5. **Envie suas alterações**
   Faça um push da sua branch para o seu fork no GitHub.

   ```sh
   git push origin feature/NomeDaSuaFeature
   ```

6. **Abra um Pull Request**
   Vá para o repositório original e abra um Pull Request da sua branch para começar a discussão sobre suas contribuições.

## Diretrizes de Pull Request

- Certifique-se de que sua branch está baseada na mais recente versão da branch principal.
- Verifique se o projeto funciona corretamente após suas alterações.
- Atualize a documentação, se necessário.
- Use mensagens de commit claras e descritivas.
- Inclua comentários em seu código, se necessário.

## Código de Conduta

Pedimos a todos os contribuidores que leiam e sigam nosso [Código de Conduta](CODE_OF_CONDUCT.md).

## Dúvidas?

Se tiver dúvidas ou precisar de ajuda, não hesite em abrir uma issue ou entrar em contato com um dos mantenedores do projeto.

Agradecemos sua contribuição e estamos ansiosos para ver suas ideias!
